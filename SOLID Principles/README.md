# SOLID Principles in Java

This folder contains examples and explanations for each of the five SOLID principles.

## Principles:
1. **SRP - Single Responsibility Principle**: A class should have only one reason to change.
2. **OCP - Open/Closed Principle**: Software entities should be open for extension but closed for modification.
3. **LSP - Liskov Substitution Principle**: Objects of a superclass should be replaceable with objects of its subclasses without breaking the application.
4. **ISP - Interface Segregation Principle**: No client should be forced to depend on methods it does not use.
5. **DIP - Dependency Inversion Principle**: High-level modules should not depend on low-level modules. Both should depend on abstractions.
